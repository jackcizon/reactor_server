# Security Policy

## Supported Code

This project does not use formal version numbers.  
All code in the repository is actively maintained, and security fixes are applied promptly when issues are discovered.

---

## Reporting a Vulnerability

We take security issues seriously. If you discover a vulnerability, please follow these steps:

1. **Contact:** Send a detailed report to me at **jack20021213cn@gmail.com**.
    - Include a description of the issue, steps to reproduce it, and any relevant files or logs.
    - Do **not** publicly disclose the issue until it has been addressed.

2. **Acknowledgment:** You will receive a confirmation within **48 hours**.

3. **Resolution:**
    - We will investigate and prioritize the fix based on severity.
    - Once fixed, a patch or update will be applied to the repository.

4. **Disclosure:**
    - Public disclosure will happen only after a fix is available.
    - You may be credited as the reporter if you wish.

---

## Additional Security Considerations

- Keep dependencies up-to-date and monitor for security advisories.
- Use secure authentication methods and environment variables where applicable.
- Report any suspicious activity related to the project to the security contact above.
